# M2 광의통화 대시보드 서비스 소개

> 이름: 엄기영 | 학번: K2023510

## 1. 서비스 개요

**M2 광의통화 대시보드**는 한국은행 경제통계시스템(ECOS)에서 제공하는 M2(광의통화) 공공데이터를 수집·시각화하는 웹 서비스입니다.

- **배포 URL:** https://m2-dashboard-sigma.vercel.app
- **GitHub:** https://github.com/planze-kee/m2_dashboard

Google OAuth 로그인만으로 누구나 대시보드에 접근할 수 있으며, 방명록 댓글 기능을 통해 사용자 간 의견을 남길 수 있습니다.

---

## 2. M2(광의통화)란?

M2(광의통화, Broad Money)는 한 나라의 통화량을 측정하는 핵심 지표입니다.

| 구분 | 포함 항목 |
|------|-----------|
| M1(협의통화) | 현금 + 요구불예금 + 수시입출금식 저축성예금 |
| M2(광의통화) | M1 + 2년 미만 정기예적금 + 머니마켓펀드(MMF) 등 |

M2는 한국은행의 통화정책 수립 및 경기 분석의 핵심 지표로 활용됩니다.
2025년 12월 한국은행 통화지표 개편으로 통계표 코드가 변경(101Y004 → 161Y011)되었으며, 본 서비스는 변경된 코드를 반영합니다.

## 4. 주요 기능

### 4.1 Google OAuth 인증
- Google 계정을 통한 소셜 로그인
- NextAuth.js 기반 JWT 세션 관리
- 로그인한 모든 Google 계정 사용자 접근 허용

### 4.2 M2 데이터 시각화
- 최근 3년간 월별 M2 광의통화 잔액 추이 라인 차트
- 최신 M2 잔액(조원), 전월 대비·전년 동월 대비 증감률 요약 카드
- 월별 상세 데이터 테이블 (최신 순 정렬)

### 4.3 ECOS API 연동 및 DB 캐싱
- `StatisticItemList` API로 항목코드·주기·최신 데이터 기간 동적 조회
- `StatisticSearch` API로 실시간 M2 데이터 수집
- 조회 결과를 Turso DB `m2_cache` 테이블에 자동 캐싱
- ECOS API 장애 시 캐시 데이터 자동 사용(폴백)

### 4.4 방명록(댓글) 기능
- 로그인한 사용자 누구나 대시보드 하단에 댓글 작성 가능
- Google 프로필 이미지, 이름, 작성 시각 표시
- 최대 300자 제한, 최신 순 정렬
- Turso DB `comments` 테이블에 영구 저장

## 5. 시스템 아키텍처

```
사용자 브라우저
    │
    ▼
Vercel (Next.js 14 App Router)
    ├── 미들웨어: /dashboard 접근 시 JWT 토큰 유무 검증 (로그인 여부만 확인)
    ├── 인증: NextAuth.js + Google OAuth 2.0
    ├── API Route /api/m2
    │       ├── 1. StatisticItemList → 항목코드/기간 조회
    │       ├── 2. StatisticSearch  → M2 월별 데이터 수집
    │       └── 3. Turso DB 캐싱 (장애 시 캐시 사용)
    ├── API Route /api/comments
    │       ├── GET  → 댓글 목록 조회
    │       └── POST → 댓글 작성 (로그인 필요)
    └── Turso DB (libSQL Edge)
            ├── allowed_users (사용자 목록)
            ├── m2_cache (M2 데이터 캐시)
            └── comments (방명록 댓글)
```

- **통계표:** `161Y011` - M2 경제주체별 보유현황 (말잔, 계절조정계열)

## 8. DB 스키마

```sql
-- 접근 허용 사용자 목록
CREATE TABLE allowed_users (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  email      TEXT    UNIQUE NOT NULL,
  name       TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- M2 데이터 캐시
CREATE TABLE m2_cache (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  period     TEXT    UNIQUE NOT NULL,  -- YYYYMM
  value      REAL    NOT NULL,         -- 십억원
  unit       TEXT    DEFAULT '십억원',
  fetched_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 방명록 댓글
CREATE TABLE comments (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  user_email TEXT    NOT NULL,
  user_name  TEXT    NOT NULL,
  user_image TEXT,
  content    TEXT    NOT NULL,         -- 최대 300자
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```
