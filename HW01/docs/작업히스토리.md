## 작업 순서

### 1단계 — 프로젝트 초기 구성

- Next.js 14 App Router 프로젝트 생성
- Tailwind CSS, Recharts 설치
- Turso DB(libSQL) 연동: `@libsql/client` 설치 및 `lib/db.ts` 작성
- DB 스키마 설계 및 `scripts/init-db.ts` 작성
  - `allowed_users` 테이블 (접근 허용 사용자)
  - `m2_cache` 테이블 (M2 데이터 캐시)

### 2단계 — Google OAuth 인증

- Google Cloud Console에서 OAuth 2.0 클라이언트 생성
- NextAuth.js v4 설정: `lib/auth.ts`, `app/api/auth/[...nextauth]/route.ts`
- JWT 세션 전략 적용
- 로그인 페이지(`app/page.tsx`) 구현: Google 로그인 버튼
- `middleware.ts`로 `/dashboard` 접근 보호

### 3단계 — ECOS API 연동

- 한국은행 ECOS API 키 발급
- `lib/ecos.ts` 작성
  - `StatisticItemList` API: 항목코드·최신 데이터 기간 동적 조회
  - `StatisticSearch` API: 월별 M2 데이터 수집
- 2025년 12월 한국은행 통화지표 개편 대응: 통계표 코드 `101Y004` → `161Y011` 변경
- `app/api/m2/route.ts` 작성: ECOS 조회 → Turso 캐싱 → 장애 시 캐시 폴백

### 4단계 — 대시보드 UI 구현

- `app/dashboard/page.tsx`: M2 데이터 대시보드 페이지
- `components/M2Chart.tsx`: Recharts 기반 라인 차트 (최근 3년 월별 추이)
- `components/Header.tsx`: 헤더 컴포넌트 (로그아웃 버튼 포함)
- 요약 카드: 최신 M2 잔액, 전월 대비 증감률, 전년 동월 대비 증감률
- 월별 상세 데이터 테이블 (최신 순 정렬)

### 5단계 — Vercel 배포

- Vercel 프로젝트 생성 및 GitHub 연동 (자동 배포 설정)
- 환경변수 설정: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `NEXTAUTH_SECRET`, `NEXTAUTH_URL`, `TURSO_DATABASE_URL`, `TURSO_AUTH_TOKEN`, `ECOS_API_KEY`
- Google OAuth 리디렉션 URI에 Vercel 도메인 추가

### 6단계 — 접근 정책 변경 (allowlist → 전체 공개)

**배경:** 초기에는 `allowed_users` 테이블에 등록된 사용자만 접근 가능하도록 구현했으나, Google 계정이 있으면 누구나 접근 가능하도록 변경

**수정 위치 3곳:**
1. `middleware.ts`: `isAllowed` 체크 제거 → JWT 토큰 유무만 확인
2. `lib/auth.ts`: JWT 콜백에서 `isEmailAllowed` DB 조회 제거
3. `app/api/m2/route.ts`: 403 응답 로직 제거 → 세션 유무만 확인
4. `app/page.tsx`: `isAllowed` 체크 제거 → 인증된 사용자 전원 `/dashboard`로 이동

**트러블슈팅:**
- Vercel 배포 후에도 "접근 권한 없음" 에러 → 위 3곳 모두 수정 필요했음
- `lib/auth.ts`의 `session` 콜백에서 미사용 `token` 파라미터 → 제거로 경고 해소

## 트러블슈팅 모음

| 문제 | 원인 | 해결 |
|------|------|------|
| 민감 파일 staged | `client_secret*.json`, `구글 Oauth 키.txt` 실수로 staged | `git restore --staged` 후 `.gitignore`에 패턴 추가 |
| SSH push 실패 | Claude 셸에 SSH 에이전트 미초기화 | `ssh-add --apple-load-keychain`으로 macOS 키체인에서 로드 |
| npm not found | nvm 환경 미로드 | `export NVM_DIR=... && . "$NVM_DIR/nvm.sh"` 선행 실행 |
| 배포 후 접근 권한 없음 | allowlist 체크가 3곳에 분산 | `middleware.ts`, `app/page.tsx`, `api/m2/route.ts` 전부 수정 |
| "데이터 로딩 실패" | `api/m2/route.ts`의 `isAllowed` 체크 누락 | 해당 403 응답 로직 제거 |
| ECOS API 오류 | 2025년 12월 통화지표 개편으로 통계표 코드 변경 | `101Y004` → `161Y011`로 업데이트 |

## 최종 DB 스키마

```sql
CREATE TABLE allowed_users (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  email      TEXT    UNIQUE NOT NULL,
  name       TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE m2_cache (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  period     TEXT    UNIQUE NOT NULL,
  value      REAL    NOT NULL,
  unit       TEXT    DEFAULT '십억원',
  fetched_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE comments (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  user_email TEXT    NOT NULL,
  user_name  TEXT    NOT NULL,
  user_image TEXT,
  content    TEXT    NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```
